# Full Research Design Workflow

Use this file when the user wants a full research design, proposal plan, thesis plan, or systematic revision.

Core chain:

`topic -> research question -> literature-review design -> causal explanation -> hypotheses -> variables -> operationalization -> hypothesis testing -> thesis structure`

## Non-Template Rule

Do not fill the workflow by importing a mechanism from another topic or example.

Each design must be rebuilt from:

1. the user's outcome Y;
2. the selected cases and time period;
3. observed variation or anomaly;
4. available evidence;
5. plausible actors and mechanisms in that context.

## 1. Topic

A topic is an entry point, not a research question.

Specify:

- phenomenon;
- actors or cases;
- time period;
- spatial scope;
- variation, change, or anomaly;
- academic or practical value;
- available evidence.

Output:

| Item | Draft |
|---|---|
| Initial topic | |
| Concrete phenomenon | |
| Actors/cases | |
| Time frame | |
| Variation/anomaly | |
| Evidence availability | |
| Research value | |

Diagnostic questions:

- Is this a field label or a concrete phenomenon?
- Who or what is the unit of analysis: state, leader, organization, firm, alliance, event, dyad, region, or institution?
- Is there variation to explain, or only a description to write?
- Is the time period narrow enough for evidence collection?
- Can the user name at least three plausible sources of evidence?
- Does the topic have academic value beyond being current or important?

Common mistakes:

- Treating a broad field as a topic, such as "external pressure", "global governance", or "regional cooperation".
- Choosing a topic that is only a single fact with no explanatory leverage.
- Choosing a fashionable topic but lacking a research puzzle.
- Choosing a normatively attractive topic but not an empirical question.
- Ignoring data access until after the theory is built.

## 2. Research Question

A good research question is clear, meaningful, and answerable.

Useful forms:

- Why did Y occur in some cases but not others?
- Why did Y change over time?
- How does X affect Y?
- Under what conditions does X produce Y?
- Why does the same pressure lead to different outcomes?

Output:

| Item | Draft |
|---|---|
| Puzzle | |
| Research question | |
| Outcome Y | |
| Cases/time frame | |
| Question type | why / how / under what conditions / with what effect |
| Evidence needed | |

Diagnostic questions:

- What exactly is the outcome to be explained?
- Does the question ask why, how, or under what conditions, rather than merely what happened?
- Can the question be answered with evidence rather than opinion?
- Is the question too broad for one thesis or article?
- Does the question identify a contrast, change, anomaly, or debate?
- Would the answer add knowledge, or only summarize known facts?

Common mistakes:

- Writing a title instead of a question.
- Asking a moral or policy preference question that cannot be empirically tested.
- Asking "what is X" when the project needs a causal or explanatory question.
- Hiding the dependent variable.
- Combining several research questions into one overloaded sentence.

## 3. Literature-Review Design

Organize user-provided literature into explanation types and debate structure.

Do not invent sources. Do not pretend to search.

Matrix:

| Source | Question | Explanation | Method/evidence | Scope | Limitation | Use |
|---|---|---|---|---|---|---|
| | | | | | | |

Diagnostic questions:

- Which explanations dominate the existing literature?
- What mechanism does each explanation imply?
- What evidence does each source use?
- What does each explanation fail to explain?
- Does the user's project extend, challenge, refine, or integrate existing explanations?
- Are the sources grouped by debate rather than by author order?

Common mistakes:

- Writing an annotated bibliography instead of a literature review.
- Saying "research is insufficient" without specifying the insufficiency.
- Creating a fake gap by ignoring relevant literature.
- Mixing literature on the broad topic with literature on the actual research question.
- Inventing citations or filling missing sources with model-generated references.

Review paragraph pattern:

```text
One group of studies explains Y through [explanation type]. This work highlights [mechanism] and uses [evidence]. It is useful because [strength]. However, it leaves open [specific gap]. My project builds on this debate by [contribution].
```

## 4. Causal Explanation

Build from outcome to cause.

| Element | Draft |
|---|---|
| Outcome Y | |
| Cause X | |
| Temporal order | |
| Mechanism M | |
| Counterfactual claim | |
| Scope condition | |
| Alternative explanation | |
| Main inference risk | |

Diagnostic questions:

- Is Y defined before X is chosen?
- Does X precede Y?
- Is the mechanism a real process, or only a label?
- What actor, institution, or entity changes in each mechanism step?
- What counterfactual does the causal claim imply?
- What alternative explanation could generate the same Y?
- Is the scope condition stated when the claim does not apply universally?
- Are confounders, mediators, and colliders separated correctly?

Common mistakes:

- Saying "X affects Y" without explaining how.
- Treating correlation as causation.
- Treating a concept label as a mechanism.
- Defining X in a way that already contains Y.
- Controlling for a mediator or collider.
- Calling an exception a scope condition without theoretical justification.

## 5. Hypotheses

A hypothesis should be testable and specific.

Formula:

```text
When X changes, Y becomes more/less likely, because mechanism M operates under condition C.
```

Table:

| Hypothesis | Direction | Condition | Mechanism | Observable implication | Disconfirming evidence |
|---|---|---|---|---|---|
| | | | | | |

Diagnostic questions:

- Does the hypothesis contain X and Y?
- Does it specify direction: increase, decrease, more likely, less likely, presence, or absence?
- Does it include the mechanism, at least briefly?
- Can evidence contradict it?
- Does it apply everywhere, or only under a condition?
- Is it different from an assumption?

Common mistakes:

- Writing a viewpoint instead of a hypothesis.
- Writing "X is related to Y" without direction.
- Making a hypothesis unfalsifiable.
- Packing several mechanisms into one hypothesis.
- Adding ad hoc exceptions after seeing inconvenient evidence.

Revision pattern:

```text
Weak: International organizations matter.
Better: When an international organization has monitoring authority, member compliance becomes more likely because information asymmetry decreases.
```

## 6. Variables

| Variable | Role | Definition | Expected role | Indicator |
|---|---|---|---|---|
| Y | Dependent variable | | | |
| X | Independent variable | | | |
| M | Mediator/mechanism | | | |
| C | Confounder/control | | | |
| D | Collider, if relevant | Common result of X and Y | Do not control casually | |

Diagnostic questions:

- Is the case being mistaken for a variable?
- Is the topic being mistaken for a variable?
- Are X and Y defined independently?
- Is M part of the mechanism rather than a competing cause?
- Is C a real confounder or just a routine control?
- Is D downstream of both X and Y in the collider pattern?

Common mistakes:

- Listing too many controls without theoretical reason.
- Treating all variables as equal rather than assigning causal roles.
- Using a mediator as a control variable.
- Using a collider as a control variable.
- Switching the meaning of X and Y across sections.

|---|---|---|---|---|---|---|
| | | | | | | |

Diagnostic questions:

- Does the indicator measure the concept or merely a convenient proxy?
- Can another researcher reproduce the coding?
- Does the indicator overlap with the outcome?
- Are missing data and source bias acknowledged?
- Is the measurement comparable across cases and time?
- Does the project need one indicator or several dimensions?

Common mistakes:

- Choosing indicators only because data are easy to find.
- Using vague concepts without dimensions.
- Using outcome evidence as the indicator for the cause.
- Hiding coding decisions.
- Ignoring validity and reliability.

## 8. Hypothesis Testing

| Hypothesis | Evidence | Method | Alternative explanation | Support criterion | Disconfirming evidence |
|---|---|---|---|---|---|
| | | | | | |

Diagnostic questions:

- What evidence should exist if the hypothesis is true?
- What evidence should be absent if it is false?
- What would the alternative explanation predict?
- Why are the cases or data appropriate?
- How are confounders handled?
- Is uncertainty reported honestly?

Common mistakes:

- Looking only for confirming evidence.
- Selecting cases only because they support the argument.
- Treating chronological narrative as process tracing.
- Treating statistical significance as proof of mechanism.
- Ignoring negative cases or inconvenient facts.

## 9. Thesis Structure

1. Introduction: puzzle, question, argument, method, contribution.
2. Literature review: explanation types, debate map, gap.
3. Theory: concepts, causal mechanism, hypotheses.
4. Research design: cases/data, variables, operationalization, method.
5. Empirical analysis: evidence organized around hypotheses or mechanism steps.
6. Conclusion: answer the question, explain contribution and limits.

Diagnostic questions:

- Does the introduction state the question early?
- Does the literature review lead to a gap rather than drift through authors?
- Does the theory section produce hypotheses?
- Does the method section explain how the hypotheses will be tested?
- Are empirical chapters organized by claims, hypotheses, or mechanism steps?
- Does the conclusion directly answer the research question?

Common mistakes:

- Starting with broad background and delaying the question.
- Writing literature review and theory as separate worlds.
- Making empirical chapters descriptive.
- Repeating the introduction in the conclusion.
- Claiming broader generalization than the design supports.

## Minimal Response Rule

If the user gives only a broad topic, do not fabricate a full design. Produce:

1. three to five possible dependent variables/outcomes;
2. two or three possible research questions;
3. the recommended question, with reasons;
4. what information is still needed;
5. a warning that mechanisms cannot be chosen until the outcome and scope are fixed.
