# Full Research Design Workflow

Use this file when the user wants a complete research design, proposal, thesis plan, or systematic revision. It expands the full chain:

**选题 -> 研究问题 -> 文献综述设计 -> 因果解释 -> 假设 -> 变量 -> 操作化 -> 假设检验 -> 论文结构**

This workflow is methodological. It does not search literature or invent citations. For literature sections, organize only user-provided titles, abstracts, notes, bibliographies, or excerpts.

## 1. 选题

选题不是论文题目，而是研究的入口。先把兴趣范围压缩到一个可研究对象。

### 选题应说明

- 现象：研究什么国际关系现象。
- 主体：国家、国际组织、领导人、企业、联盟、制度、地区或事件。
- 时间：哪一段时期。
- 空间：哪个国家、地区、双边关系或国际体系层次。
- 变化或差异：发生了什么变化，或不同案例为什么不同。
- 学术或现实意义：为什么值得研究。

### 好选题标准

- 不只是大主题，如“中美关系”“国际秩序”“一带一路”。
- 有可观察现象，如政策变化、合作成败、冲突升级、联盟选择、制度效果。
- 能形成比较或解释，如前后变化、案例差异、预期与事实不一致。
- 有资料可获得，如官方文件、统计数据、新闻档案、演讲文本、二手研究、案例材料。

### 常见问题

- 题目太大：把一个领域当成题目。
- 题目太小：只描述一个事实，没有解释空间。
- 题目太空：概念漂亮但没有可观察对象。
- 题目太热：现实意义强，但学术问题不清楚。

## 2. 研究问题

研究问题是整篇论文的发动机。它要问一个能够用证据回答的问题。

### 从主题到问题

把“研究 X”改成：

- 为什么 Y 在某些情况下发生，而在另一些情况下没有发生？
- 为什么同样的 X 在不同案例中导致不同 Y？
- X 通过什么机制影响 Y？
- 在什么条件下 X 更可能导致 Y？
- 为什么某项政策、制度或行为在某一时期发生变化？

### 好问题的三项标准

- **明确真实**：指向具体、可观察的现象。
- **有意义**：能回应理论争论、经验异常或现实难题。
- **能回答**：有资料、有方法、有边界。

### 必备元素

- 因变量或结果：你要解释什么。
- 解释类型：原因、机制、条件、差异、变化。
- 案例范围：研究哪些对象。
- 时间范围：何时发生。
- 比较基准：与什么相比。

### 常见错误

- 问题其实是口号：“如何推动世界和平？”
- 问题只是描述：“什么是数字外交？”
- 问题没有结果变量：“研究大国竞争中的技术问题。”
- 问题无法证伪：“国家利益是否重要？”

| 项目 | 内容 |

| 研究问题 | |
| 因变量 Y | |
| 解释对象 | |
| 案例/时间范围 | |
| 问题类型 | why / how / under what conditions / with what effect |
| 可回答性证据 | |

## 3. 文献综述设计

文献综述设计不是替用户找文献，而是把用户已有文献、摘要、书目或读书笔记组织成争论地图。

### 文献综述应回答

- 这个问题已有哪几类解释？
- 每类解释的核心机制是什么？
- 它们使用了什么证据和方法？
- 它们解释了什么，解释不了什么？
- 用户的研究补充哪一个缺口？

### 组织方式

优先按解释逻辑组织，而不是按作者逐篇排列：

- 权力结构解释。
- 国内政治解释。
- 制度主义解释。
- 观念/认知解释。
- 经济相互依赖解释。
- 历史路径解释。
- 方法或证据路径差异。

### 文献矩阵

| 已提供材料 | 研究问题 | 核心解释 | 方法/证据 | 适用范围 | 局限 | 本文如何使用 |
|---|---|---|---|---|---|---|

### 缺口类型

- 现象缺口：已有研究没有解释某个新现象或异常案例。
- 机制缺口：已有研究说 X 影响 Y，但没有说明如何影响。
- 条件缺口：已有研究没有说明 X 在什么条件下有效。
- 测量缺口：已有研究概念和指标不够清楚。
- 证据缺口：已有研究证据不足或案例选择有偏。
- 整合缺口：两个研究传统可以对话但尚未连接。

### 常见错误

- 逐篇复述，没有争论结构。
- 只说“研究不足”，不说不足在哪里。
- 综述和自己的研究问题脱节。
- 编造或补全不存在的文献出处。

## 4. 因果解释

因果解释回答“为什么”。它不是简单说 X 影响 Y，而是说明 X 如何、何时、通过什么机制导致 Y。

### 因果解释的构成

- X：原因或关键解释变量。
- Y：结果或被解释变量。
- M：因果机制。
- S：范围条件。
- R：竞争性解释。
- O：可观察含义。

### 机制链模板

```text
X 改变了行为体的认知、激励、资源或约束。
行为体因此采取某种策略或行为。
该行为改变互动结构、制度安排或资源分配。
最终产生 Y。
```

### 机制检查

- 时间顺序是否清楚？
- 每一步是否有行为体？
- 每一步是否能留下证据？
- 机制是否区别于竞争性解释？
- 机制是否适用于选定案例范围？

### 常见错误

- 把相关性当因果。
- 把概念堆叠当机制。
- 只有宏观结构，没有行动过程。
- 没有竞争性解释。
- 解释范围过大。

## 5. 假设

假设是因果解释的可检验表达。它必须比“观点”更精确。

### 好假设应包含

- 自变量 X。
- 因变量 Y。
- 方向或差异。
- 条件或范围。
- 简要机制。
- 可观察含义。

### 假设形式

```text
H1：在 S 条件下，X 越强，Y 越可能发生，因为 M。
H2：当 S 不存在时，X 对 Y 的影响减弱或不成立。
H3：如果 M 成立，应观察到 O1、O2、O3。
```

### 竞争性假设

至少列出一个竞争性解释：

- 结构性解释。
- 国内政治解释。
- 制度解释。
- 观念认知解释。
- 历史路径解释。
- 领导人或官僚政治解释。

### 常见错误

- 假设只是判断：“国际组织很重要。”
- 假设没有方向：“X 与 Y 有关系。”
- 假设不可观察。
- 假设和研究问题不对应。
- 假设过多但没有主线。

## 6. 变量

变量是可以变化、比较和观察的概念。变量设计要服务于研究问题和因果解释。

### 变量类型

| 类型 | 作用 | 例子 |
|---|---|---|
| 因变量 Y | 被解释的结果 | 冲突是否复发、政策是否转向、合作是否达成 |
| 自变量 X | 主要原因 | 威胁感知、经济依赖、制度约束、权力转移 |
| 中介变量 M | 传导机制 | 承诺可信度、资源重组、联盟协调 |
| 调节变量 S | 改变关系强弱 | 制度化程度、危机强度、政体类型 |
| 混杂变量 C | 同时影响 X 和 Y | 地理距离、经济规模、历史敌意 |
| 控制变量 | 用来排除竞争解释 | 能力差距、贸易规模、共同盟友 |

### 变量表

| 变量 | 类型 | 概念定义 | 预期作用 | 与机制关系 | 可能指标 |
|---|---|---|---|---|---|

### 常见错误

- 把案例当变量。
- 把主题当变量。
- 因变量和自变量互相定义。
- 控制变量没有理论理由。
- 中介变量被误当控制变量。

## 7. 操作化

操作化把抽象概念变成可观察、可编码、可比较的指标。

### 操作化步骤

1. 给出概念定义。
2. 拆分维度。
3. 为每个维度选择指标。
4. 说明数据来源。
5. 写出编码规则。
6. 评估效度和信度。
7. 说明局限和稳健性检查。

### 指标选择标准

- 是否能真实反映概念。
- 是否能跨案例或跨时间比较。
- 是否有可靠来源。
- 是否避免与因变量重叠。
- 是否能捕捉变化。

### 操作化表

| 概念 | 定义 | 维度 | 指标 | 数据来源 | 编码规则 | 效度风险 | 信度检查 |
|---|---|---|---|---|---|---|---|

### 常见错误

- 用一个模糊词解释另一个模糊词。
- 指标只是因为容易获得。
- 指标直接等于结论。
- 没有说明数据缺失和偏误。
- 没有编码规则。

## 8. 假设检验

假设检验不是“找支持材料”，而是比较主假设与竞争性解释哪一个更能解释事实。

### 检验设计应说明

- 要检验哪个假设。
- 需要哪些证据。
- 证据如何支持或削弱假设。
- 如何处理竞争性解释。
- 如何选择案例或样本。
- 如何控制混杂变量。
- 什么证据会使假设失败。

### 定性检验

适合案例少、机制复杂、证据类型多的研究：

- 单案例研究。
- 比较案例研究。
- 过程追踪。
- 反事实分析。
- 最相似案例比较。
- 最不同案例比较。

### 定量检验

适合样本较多、指标可量化、关系可系统比较的研究：

- 明确因变量和自变量。
- 解释样本来源和时间范围。
- 说明控制变量。
- 报告稳健性和替代模型。
- 不把统计显著性等同于因果机制成立。

### 检验表

| 假设 | 可观察含义 | 证据/数据 | 方法 | 竞争解释 | 支持标准 | 反驳证据 |
|---|---|---|---|---|---|---|

### 常见错误

- 只找支持材料。
- 案例选择只看因变量。
- 忽视负面证据。
- 把描述性叙事当检验。
- 控制了中介变量导致过度控制。

## 9. 论文结构

论文结构应服务于研究问题，而不是材料堆放顺序。

### 标准结构

1. **导论**
   - 研究困惑。
   - 研究问题。
   - 研究意义。
   - 核心论点。
   - 方法和证据预告。
2. **文献综述设计**
   - 既有解释类型。
   - 争论地图。
   - 研究缺口。
3. **理论框架**
   - 概念定义。
   - 因果机制。
   - 假设。
4. **研究设计**
   - 案例/样本选择。
   - 变量与操作化。
   - 检验方法。
   - 竞争解释和控制策略。
5. **经验检验**
   - 按机制步骤或假设组织证据。
   - 比较主解释和竞争解释。
6. **结论**
   - 回答研究问题。
   - 说明理论贡献和现实含义。
   - 承认局限。

### 结构检查

- 导论是否直接提出问题？
- 文献综述是否导向缺口？
- 理论部分是否产生假设？
- 方法部分是否说明如何检验？
- 经验部分是否围绕假设而不是按时间流水账？
- 结论是否回到研究问题？

## Minimal Response Rule

When the user gives only a broad topic, do not produce a full fake design immediately. First produce:

1. Three possible research questions.
2. The recommended one.
3. Why it is answerable.
4. What information is still needed.

When the user gives a draft, diagnose the weakest link in the chain and revise that link first.
