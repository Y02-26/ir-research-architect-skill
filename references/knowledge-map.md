# Knowledge Map

This knowledge base is an OCR-assisted, paraphrased synthesis of a Chinese-language textbook on practical international relations research methods. It is designed for research-method assistance, not for reproducing the book.

Do not quote long passages. Do not claim page precision unless the original source has been checked.

## OCR And Source Caveats

- The source was a scanned PDF; OCR may distort terms, names, and page numbers.
- Treat these files as curated methodological notes, not as verbatim authority.
- If exact wording or citation is required, consult the original book.
- The repository does not include the original PDF or full book text.

## Book-Derived Research Sequence

Use this sequence as the default workflow:

1. Nature of IR research: distinguish academic research, policy research, diplomatic practice, and commentary.
2. Scientific method: understand research quality, iteration, evidence, scientific spirit, and limits of IR inquiry.
3. Research question selection: move from puzzle to question; make the question clear, meaningful, and answerable.
4. Literature-review design: organize user-provided literature into debates, explanations, methods, and gaps.
5. Causal explanation: clarify causal relation, causal diagrams, counterfactuals, mechanisms, and causal hypotheses.
6. Operationalization and measurement: translate concepts into indicators and assess validity/reliability.
7. Hypothesis testing: choose evidence, cases/data, controls, and qualitative/quantitative testing strategies.
8. Academic writing: organize the paper or proposal into question, literature, theory, method, evidence, and conclusion.

## Which Reference To Use

Use this as an input-responsive retrieval map. Select by the user's actual request:

- User gives a broad topic or asks "what can I study": open `reasoning-patterns.md` and `question-and-literature.md`.
- User gives a draft title, paragraph, proposal, or asks "what is wrong": open `checklists.md`; add the specific reference for the weakest link.
- User asks about literature review design from provided materials: open `question-and-literature.md`.
- User asks about causal mechanism, hypotheses, scope condition, confounder, mediator, or collider: open `causal-explanation.md`.
- User asks about variable definition, indicators, coding, validity, or reliability: open `operationalization-measurement.md`.
- User asks about case selection, process tracing, controls, quantitative tests, or evidence: open `hypothesis-testing.md`.
- User asks about introduction, proposal, thesis outline, chapter structure, or writing: open `thesis-writing.md`.
- User asks in Chinese or wants Chinese academic phrasing: open `chinese-output-style.md`.
- User asks whether the project is academic, policy, or mixed: open `research-orientation.md`.

Do not load the full workflow unless the user asks for an end-to-end design.

## Default Output Patterns

- Research-design table: question, puzzle, Y, X, mechanism, scope, cases, evidence, method.
- Full workflow table: topic, question, literature-review design, causal explanation, hypotheses, variables, operationalization, testing, thesis structure.
- Causal explanation map: X -> mechanism -> Y, with confounders, colliders, counterfactuals, and alternatives.
- Literature-material matrix: source, question, theory, method, evidence, finding, limitation, relevance.
- Operationalization table: concept, definition, dimensions, indicators, source, coding rule, validity risk.
- Testing plan: hypothesis, observable implication, evidence, method, alternative explanation, falsification risk.

For broad topics, prefer generated outcome-discovery outputs before full designs:

- candidate dependent variables/outcomes generated from the user's wording;
- possible variation or anomaly;
- candidate research questions;
- missing scope/evidence information.
